Final Cut Pro X and Dropbox Part 3 - Summary of Results

Oct 3, 2014
1 minute read

My experiment with using Final Cut Pro X and Dropbox as a collaborative tool has yielded some educational results. I have learned how a SparseDisk segments it’s innards revealing its (dir) nature. I also see some potential and sure fire ways to get hurt.

The few snafus along the way have not necessarily been a dropbox nor FCP issue exactly, it was how these programs work with there own files. If the project has compound clips created within it, the new version of Final Cut Pro X now a adds said compound to your events. I don’t know exactly where this compound lives. (at this time I think it resides in the render files folder)

Restoring

The restorative properties of dropbox As it applies to a Sparse disk file. I had a theory that whatever changes I made in the DB folder would be able to roll back in time much like other files in the DB universe. I was wrong. Seem that sparsedisk is seen as a Folder to dropbox. Thus a previous state doesn’t exist. (TimeMachine can if needed)
